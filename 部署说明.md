# 云服务器部署说明

## 推荐服务器配置（以 Vultr 为例）

| 项目 | 推荐选择 |
|------|---------|
| 套餐 | Cloud Compute - Regular Performance，$6/月（1 vCPU / 1GB RAM / 25GB SSD） |
| 系统 | Ubuntu 22.04 LTS x64 |
| 地区 | Tokyo 或 Singapore（香港用户延迟最低） |

> 1GB 内存是最低要求，Chromium 无头浏览器比较吃内存。如果发现频繁崩溃，升级到 2GB（$12/月）。

---

## 一键安装

> **必须先下载再运行**，不能用 `curl | bash`（交互式密码输入需要终端）

```bash
curl -fsSL https://raw.githubusercontent.com/ProphetKL/whatsapp-bot/main/install.sh -o install.sh && sudo bash install.sh
```

脚本自动完成以下所有步骤：

1. 检查系统（仅支持 Ubuntu / Debian）
2. **强制设置管理界面用户名和密码**（密码至少 8 位，两次确认）
3. 安装 Chromium 所需系统依赖
4. 检查并安装 Node.js 20+
5. 安装 PM2 进程管理器
6. 从 GitHub 下载项目代码
7. 安装 npm 依赖
8. 写入 `.env` 配置文件（权限 600，仅 root 可读）
9. 安装 `whatsapp-upgrade` 升级命令
10. 启动机器人（PM2 守护）
11. 配置开机自启
12. 开放防火墙端口

安装完成后，终端会显示管理界面地址、用户名，以及常用命令。

---

## 安装后使用

1. 浏览器打开 `http://服务器IP:3000`
2. 输入安装时设置的用户名和密码
3. 扫描页面上的 QR 码，完成 WhatsApp 登录
4. 登录成功后 QR 码消失，显示任务管理界面
5. 点击「新增任务」，填写群组名称、消息内容和发送时间

---

## 升级到最新版本

```bash
sudo whatsapp-upgrade
```

该命令会自动：拉取 GitHub 最新代码 → 更新 npm 依赖 → 重启机器人。

---

## 常用 PM2 命令

```bash
pm2 logs whatsapp-bot      # 查看实时日志
pm2 status                 # 查看运行状态
pm2 restart whatsapp-bot   # 重启
pm2 stop whatsapp-bot      # 停止
```

---

## 安全说明

- 管理界面强制 Basic Auth，**未设置密码时程序拒绝启动**
- 同一 IP 15 分钟内登录失败 5 次后自动封锁，防止暴力破解
- `.env` 文件权限 600，仅 root 可读
- 建议配合 Nginx 反向代理 + HTTPS，防止密码明文传输

---

## 注意事项

- `data/` 目录保存 WhatsApp 登录会话，**不要删除**，否则需重新扫码
- 服务器重启后 PM2 自动重启机器人（安装时已配置开机自启）
- 遇到 "detached Frame" 错误时程序会自动重连，无需手动干预
- 仅支持个人 WhatsApp 账号（不支持 WhatsApp Business API）

---

## 防火墙（手动配置）

安装脚本已自动开放端口。如需手动操作：

```bash
sudo ufw allow 3000/tcp
sudo ufw enable
```
